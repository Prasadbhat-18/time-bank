📊 STORAGE HIERARCHY (How Services Are Saved)
─────────────────────────────────────────────────────────────────────────────
When you post a service:

1. ☁️  Firebase Firestore (PRIMARY - Cloud, cross-device)
2. 💾 localStorage (SECONDARY - Browser's local storage)
3. 🔒 permanentStorage (TERTIARY - Survives logout)
4. 📦 sharedStorage (QUATERNARY - Cross-session backup)

❓ TROUBLESHOOTING
─────────────────────────────────────────────────────────────────────────────
Q: Services still not visible on other device?
A: 1. Check same account on both devices
   2. Check internet connection
   3. Clear browser cache (Ctrl+Shift+Delete)
   4. Hard refresh (Ctrl+Shift+R)
   5. Check Firebase credentials in .env.local

Q: Console shows Firebase error?
A: 1. Verify .env.local has valid Firebase config
   2. Check Firestore is enabled in Firebase console
   3. Check Firestore rules allow read/write
   4. Try hard refresh: Ctrl+Shift+R

Q: Services visible locally but not on other devices?
A: 1. Ensure logged in with same account
   2. Check internet connection
   3. Wait 5-10 seconds for Firebase sync
   4. Refresh the page
